# V2 GigE PS HLS FPGA Project

V2 moves from USB reference input to a self-built dual GigE IR camera system while keeping PS Linux responsible for capture:

```text
2 x MV-CS013-60GN GigE IR cameras
-> Gigabit switch / PS Ethernet
-> PS Linux capture
-> PS DDR buffers
-> AXI DMA / AXI4-Stream
-> shared HLS stereo_pipeline IP
-> depth stream + pointcloud stream
```

Camera trigger timing and the infrared laser/speckle projector should be generated by PL, even though image capture remains in PS Linux:

```text
PS Linux AXI-Lite configuration
-> PL trigger_controller
-> left_camera_trigger
-> right_camera_trigger
-> laser/projector_enable
```

## Project Assumptions

- Board: AXU5EVB-E, XCZU5EV-2SFVC784I.
- Tools: Vivado/Vitis/Vitis HLS 2021.2.
- Cameras: two MV-CS013-60GN industrial cameras.
- Camera format: `1224 x 1024 Mono8 @ 30 fps`.
- Camera interface: GigE Vision 2.0 / GenICam over Gigabit Ethernet.
- Trigger: Line0 input, Line1 output, Line2 configurable I/O are reserved for synchronization and projector timing.
- PL timing control: frame period, trigger pulse width, exposure window, laser lead/lag, laser on-time, polarity, frame counter, and status counters.
- HLS IP is shared from `../../common/hls`.

## PL Synchronization Scope

V2 keeps packet/frame acquisition in PS software, but PL should own deterministic timing:

- generate simultaneous or calibrated-phase left/right camera trigger pulses;
- drive laser/projector enable with configurable lead time before exposure;
- optionally output an exposure window marker for measurement;
- count frames and trigger events for PS diagnostics;
- expose control/status registers through AXI-Lite.

The electrical interface must be designed carefully. FPGA I/O should usually drive camera trigger inputs and laser enable through level shifters, optocouplers, MOSFETs, or a dedicated driver stage. Laser/projector outputs should default to off after reset.

## First Build Steps

```bash
cd ../../common/hls
vitis_hls -f scripts/run_all_csims.tcl
vitis_hls -f scripts/export_all_ips.tcl
```

Then complete Vivado BD integration and export `vitis/xsa/v2_gige_ps_hls.xsa`.

## Open Source Code Candidates

- GigE Vision capture: Aravis or vendor MVS SDK on Linux.
- Synchronization: GenICam trigger mode examples, GPIO/line-control utilities.
- FPGA control: Xilinx AXI DMA Linux examples plus generated HLS IP drivers.

V2 should reuse the same PL/HLS algorithm IP as V1. Most new PS-side work is camera discovery, trigger setup, packet/frame validation, and DMA scheduling. The timing-critical camera/projector synchronization should be implemented in PL.
